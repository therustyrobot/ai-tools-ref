# Phase 3: AI Categorization - Context

**Gathered:** 2026-04-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Create `scripts/categorize.py` that calls the GitHub Models API in 10-repo batches (seeded with the `starred_repos.md` taxonomy) to produce `_data/categories.json`. Update `scripts/generate.py` to render the full two-level hierarchical taxonomy (top-level category → subcategory → repos) with stable anchor IDs.

This phase does NOT add search, filtering, caching, or any other new capability — only the AI categorization script and the HTML hierarchy upgrade.

</domain>

<decisions>
## Implementation Decisions

### Hierarchical Display

- **D-01:** Subcategories render as **nested sub-sections** within each top-level category section in the HTML — not flat with subcategory badges on cards. Each category section contains one or more subcategory sub-headers followed by their repo cards.
- **D-02:** Subcategory headers must be **visually distinct** from top-level category headers, using a **Safety Orange (#FF5F1F) background** with white text — creating an obvious, high-contrast divider when scrolling quickly through the page.
- **D-03:** The sidebar navigation shows **top-level categories only** (with repo counts). Subcategory sub-headers are visible inline as the user scrolls into each section, not listed in the sidebar.
- **D-04:** `generate.py` needs a new `group_by_categories_hierarchical()` function returning a nested structure `{category: {subcategory: [repos]}}`, and a new `render_subcategory_header()` (or equivalent) renderer. The existing flat `group_by_categories()` and `group_by_language()` remain for backward compatibility.

### Category Metadata in generate.py

- **D-05:** Add a **new `CATEGORY_META` dict** in `generate.py`, parallel to `LANG_META`, keyed by category slug → `(display_name, material_icon_name)`. Do NOT extend `LANG_META` with AI category slugs — keep the two dicts separate.
- **D-06:** `render_section()` (and the new subcategory renderer) should check `CATEGORY_META` first, then `LANG_META`, then fall back to `("Unknown", "category")` for any unrecognized slug.
- **D-07:** Icons are rendered as Material Icons ligature spans (already loaded in `<head>`): `<span class="material-icons">icon_name</span>`. No new CDN dependency.
- **D-08:** Full `CATEGORY_META` mapping (slug → display_name, material_icon_name):

### the agent's Discretion

- **Fallback behavior:** If `categories.json` is absent or a repo's `full_name` is not found in it, fall back to language-based grouping (existing Phase 2 behavior). This preserves the page working even when categorize.py hasn't been run.
- **Test strategy for categorize.py:** Mock the `requests.post` call in unit tests. Test all non-API logic (batch slicing, slug derivation, JSON parse/retry, output writing) with mocked responses. No real API calls in tests.
- **JSON parse error handling:** On `JSONDecodeError`, log the raw model response and retry once with a clarifying message. On second failure, log an error and assign all repos in the batch to "Other" — do not crash the pipeline.
- **Repo-to-category assignment format per AI call:** The model should return a JSON object where each key is `"owner/repo"` and each value has `{"category": "...", "subcategory": "...", "slug": "..."}`. Slug is derived by categorize.py from the canonical category name (not generated by the model) to ensure stability.

## Existing Code Insights

### Reusable Assets
- `group_by_categories(repos, cat_map)` in `generate.py` — already implemented, groups by top-level `cat_map[full_name]["category"]`. Basis for the new hierarchical grouping function.
- `load_categories(path)` in `generate.py` — already implemented, returns `{}` if absent (graceful fallback).

- `fetch_stars.py` — established pattern for GITHUB_TOKEN env var, requests pagination, _data/ file writing.

### Established Patterns
- Scripts read `GITHUB_TOKEN` from `os.environ["GITHUB_TOKEN"]` (KeyError = fail-fast).
- Output files written to `_data/` with `os.makedirs(os.path.dirname(path), exist_ok=True)`.
- Python stdlib + `requests` only — no new pip dependencies.
- Pure Python f-strings for HTML generation (no templating frameworks).

### Integration Points
- `generate.py` main entry point: `cat_map = load_categories()` → `if cat_map: group_by_categories(...)` — this branch needs to switch to the new hierarchical grouper.
- `render_section()` already calls `LANG_META.get(slug, ...)` — update lookup order to check `CATEGORY_META` first.

<specifics>
## Specific Ideas

- Subcategory headers: Safety Orange background (`bg-primary` in Tailwind / `#FF5F1F`), white text, full-width, distinct uppercase label — visually punchy enough to serve as a scrolling divider.
- Material Icons already loaded via `<link href="https://fonts.googleapis.com/icon?family=Material+Icons">` in the existing `<head>` block — zero new CDN dependency.
- Slug derivation: lowercase, hyphens, strip non-alphanumeric — same as `language_to_slug()` already does. Derive in `categorize.py` from the canonical category name returned by the model, not from the model's raw slug output (prevents model-invented slugs from breaking anchors).

</specifics>
