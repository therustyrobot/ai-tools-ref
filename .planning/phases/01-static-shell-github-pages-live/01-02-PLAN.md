  <action>
**Prerequisite check (SETUP-01):** Before enabling Pages, log a reminder that the repo must eventually be on the GitHub account that owns the starred repos for Phase 4 automation to work. Phase 1–3 development is unblocked regardless of account. Output this line to stdout:
```
NOTE (SETUP-01): Confirm repo is on the account that owns your GitHub stars before enabling Phase 4 automation. Current repo: therustyrobot/ai-tools-ref
```

**Enable GitHub Pages via `gh` CLI:**

```bash
gh api repos/therustyrobot/ai-tools-ref/pages \
  --method POST \
  -f source[branch]=main \
  -f source[path]=/docs
```

If this returns `409 Conflict` (Pages already enabled), update the source instead:
```bash
gh api repos/therustyrobot/ai-tools-ref/pages \
  --method PUT \
  -f source[branch]=main \
  -f source[path]=/docs
```

**Wait for deployment:** GitHub Pages takes 1–5 minutes after initial configuration. Poll until the URL responds:
```bash
for i in $(seq 1 12); do
  STATUS=$(curl -s -o /dev/null -w "%{http_code}" https://therustyrobot.github.io/ai-tools-ref/)
  echo "Attempt $i: HTTP $STATUS"
  if [ "$STATUS" = "200" ]; then
    echo "Pages is live!"
    break
  fi
  sleep 30
done
```

**Expected outcome:** HTTP 200 within ~5 minutes (12 attempts × 30s = 6 minutes max).

**If HTTP 404 persists after 10 minutes:**
1. Confirm `docs/index.html` is on the `main` branch: `git log --oneline -3`
2. Check Pages configuration in GitHub: `gh api repos/therustyrobot/ai-tools-ref/pages`
3. Verify the source shows `{"branch":"main","path":"/docs"}`

**Do NOT** proceed to the checkpoint until the curl poll returns HTTP 200 at least once.
